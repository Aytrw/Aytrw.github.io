Shortest Path Faster Algorithm

SPFA算法是西南交通大学段凡丁发表的求单源最短路径的一种算法
他实际上是Bell-Ford算法的改进
很多情况下我们是需要求负权边的，而Dijkstra算法却无法适用
这时我们可以使用SPFA算法

核心思想

用一个队列que不断来记录待优化的结点
然后把优化完的结点邻接的结点加入队列que
当然，初始时要将源点加入que
同时，我们用一个数组dis来记录从源点到i号点的最短路径
如果目前不可达，将其记作+∞
初始时，将源点的dis[1]初始化为0，表示距离是0

实例

首先，给定一张有向图，边上的数字代表其权值
右边的表格为dis数组，que表示当前队列
其中红色结点表示已优化完毕的结点
黑色结点表示待优化的结点
dis数组中红色数字代表dis[i]更新了

优化1号结点：相邻的是2号和6号结点，入队：
1到2的距离是dis[1]+边(1,2)的距离 = 0+1 < dis[2] = ∞
1到6的距离是dis[1]+边(1,6)的距离 = 0+4 < dis[6] = ∞
我们更新dis[2]和dis[6]，且1号结点优化完毕，出队

优化2号结点：相邻的是3号和5号结点，入队：
1到5的距离是dis[2]+边(2,3)的距离 = 1+3 < dis[5]= ∞
1到3的距离是dis[2]+边(2,5)的距离 = 1+4 < dis[3]= ∞
我们更新dis[5]和dis[3]，且2号结点优化完毕，出队

优化6号结点：没有相邻结点
6号结点直接优化完毕，出队

优化3号结点：相邻的是4号和6号结点，入队：
1到4的距离是dis[3]+边(3,4)的距离 = 5+7 < dis[4] = ∞
1到6的距离是dis[3]+边(3,6)的距离 = 5+2 > dis[6] = 4
我们更新dis[4]，且3号结点优化完毕，出队

优化5号结点：相邻的是4号和6号结点，入队：
1到4的距离是dis[5]+边(5,4)的距离 = 4+3 < dis[4] = 12
1到6的距离是dis[5]+边(5,6)的距离 = 4+2 > dis[6] = 4
我们更新dis[4]，且5号结点优化完毕，出队

依此优化4，6，4，6号结点：
由于这两个结点出度都为0，即没有指向的结点
故直接优化完毕，全部出队

最后，dis[1..6]就是我们所要的答案

链式前向星

前向星是什么

是一种介于邻接矩阵和邻接表之间的一种比较均衡的存储结构
是一种特殊的边集数组,其中每条边按照起点排序
并且记录下以某个点为起点的所有边在数组中的起始位置

前向星有什么用

对于图的存储结构,我们通常使用两种储存结构:
第一种：邻接矩阵 (Adjacency Matrix)
邻接矩阵是一种非常占用空间的储存结构
当图中有很多结点的时候,比方说23333个
就得开一个数组A[23333][23333],不爆才怪！
第二种：邻接表 (Adjacency List)
邻接表虽然不占用多大空间
但其的指针操作非常麻烦，写起来也很困难
于是我们就想出了一种较为均衡的储存结构前向星（Forward Star）
那么前向星具体有什么好处呢？
我们可以用O(1)的时间找到以i为起点的第一条边
我们也可以用O(len[i])的时间找到以i为起点的所有边
对于图的DFS,BFS,SPFA,前向星便再适合不过了
