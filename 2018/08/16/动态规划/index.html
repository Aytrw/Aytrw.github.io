动态规划

发表于 2018-08-16 | 分类于 动态规划

动态规划作为运筹学的一个分支，是让我们初学时比较头疼的算法
对于求解最优解的问题而言，我们通常要使用动态规划算法
下面，我们将详解动态规划（以下简称DP）
Q：为什么有标题图片了？A：因为重要啊！
PS：本文也许会写的比较长

DP是什么

DP是运筹学的一个分支，可以将原问题分解成许多子问题进行求解
在求解过程中，保存已解决的子问题，就可以避免无意义的重复计算
举个例子，对于计算$5!$，即5的阶乘，即$5×4×3×2×1$
假设我们已经知道了$4!$，接下来我们只要将$4!×5$就可以得到$5!$

简单的实例

我们先来看一个例子
我们给定一些起点$S_n$和终点$T_n$
每条边上的蓝字表示两结点间的距离
求任意起点到任意终点的最短路径的距离
当然，我们规定只允许往右走
对于这样的问题，我们该如何求解呢？
如果你没有学过DP，你首先应该想到的是枚举
我们将其称为暴力算法

如果使用暴力算法求解，我们可以把每一条路径列举出来
然后逐一比较，找出最小的那条即可

对于这样的算法，如果结点的个数少的话，是可行的
但是如果数据量很大，比如网络的层数有k层
那么，路径的数量也要达到$2^k$条，显然会超时
于是我们就想到了一种更优的算法——动态规划

子问题的界定

在上面的问题中，我们可以看出
对于每一个决策而言，都对应着一个子问题
刚刚我们实例中的子问题中
后边界不变，都是终点，而前边界逐渐前移
为了方便说明，我们将1，2号结点放入集合A
比如$A_1$表示1号结点，$S_2$表示起点2
类比地，我们将3，4，5放入集合B，6，7放入集合C
同时，我们也把起点和终点放入集合S和T中
也就是每个竖的一列，都把他放到一个集合里面，于是：
对于决策1而言，我们只要考虑C到T的距离
对于决策2而言，我们要考虑B，C到T的距离
对于决策3而言，我们要考虑A，B，C到T的距离
对于决策4而言，我们要考虑S，A，B，C到T的距离
这下明白了吧，所谓的后边界和前边界
对于处理这些距离的问题，就是所谓的子问题

状态转移方程

对于每一个子问题，我们都需要一个决策来解决
那么这种决策是什么，如果你理解了上面的实例，应该会有所体会
我们仍旧记最短路径为$opt[i]$
当我们在某处决策时，我们总会要挑选短的那条路
比如在$C_i$结点时，假如接下来可以走到$T_i$和$T_j$
那么我们就要考虑到$T_i$短一些，还是到$T_j$短一些
于是我们可以列出一个很关键的式子：
$$opt(C_i) = min
\begin{cases}
dist(C_i,T_i)& \text{}\\
dist(C_i,T_j)& \text{}
\end{cases}$$
这样的式子应该不难理解，根据上面的文字就可以推出
这就是我们所需要的决策，而这个式子就是决策1
但是，这只不过是对于C集合中的结点的OPT值，我们还需要其他的结点
于是，我们推导出其他结点的决策：
$$decision1:opt(C_i) = min
\begin{cases}
dist(C_i,T_i)& \text{}\\
dist(C_i,T_j)& \text{}
\end{cases}$$
$$decision2:opt(B_i) = min
\begin{cases}
dist(B_i,C_i) + opt(C_i)& \text{}\\
dist(B_i,C_j) + opt(C_j)& \text{}
\end{cases}$$
$$decision3:opt(A_i) = min
\begin{cases}
dist(A_i,B_i) + opt(B_i)& \text{}\\
dist(A_i,B_j) + opt(B_j)& \text{}
\end{cases}$$
$$decision4:opt(S_i) = min
\begin{cases}
dist(S_i,A_i) + opt(A_i)& \text{}\\
dist(S_i,A_j) + opt(A_j)& \text{}
\end{cases}$$
这样的决策是基于该结点连接着i,j两个结点的基础上
要是只连着一个结点，只需分类讨论即可
于是，我们就将这样的决策称为状态转移方程
肯定地说，这就是DP的关键所在

为什么要用DP

实际上，我们利用DP，是为了解决重叠子问题
为了加深理解，我们再来看一个例子
给定一个序列$[1，1，2，3，5，8，13，21，…]$
如果你数感比较好的话，可以轻易看出这个数列中
对于第n项($n \geq 3$)而言，其值等于n-1和n-2项之和
这样的数列我们称为斐波那契数列
于是我们可以得出斐波那契的递归式：
$$fib(n) =
\begin{cases}
1 & \text{n = 1 or n = 2}\\
fib(n-1) + fib(n-2) & \text{n ≥ 3}
\end{cases}$$
我将4带入递归式，可以得到$fib(4)$的一棵二叉树：
接下来，我再将5带入递归式：
不难看出，以$fib(3)$为根结点的子树出现了2次
这就是所谓的重叠子问题
然后如果我带入更大的数据，这棵树将会变得更加庞大
可以看出，这棵树的时间复杂度约为$O(n^2)$
那么我们怎么解决这样的重叠子问题呢？
因为$fib(3)$重复出现了，因此我们只要把$fib(3)$保存起来即可
而$fib(3)$只和$fib(2)$和$fib(1)$有关
这样只需要开一个数组$fib[1..n]$即可
把每一个需要利用的值保存到$fib[i]$中去
这样就可以瞬间解决重叠子问题，也能将时间复杂度降为$O(n)$
递推式可以是这样的：
$$fib[n] =
\begin{cases}
1 & \text{n = 1 or n = 2}\\
fib[n-1] + fib[n-2] & \text{n ≥ 3}
\end{cases}$$
当然，因为dp只和前两个状态有关，我们可以利用滚动数组降低空间复杂度
但是这里不详细讨论，因为本文的主题在于dp，而不在于dp的优化
然而，上面的两个例题只不过是DP的冰山一角
